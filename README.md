# 1. Introduction

Phishing is a cyberattack technique used by attackers to trick users into revealing sensitive information such as passwords, banking details, OTPs, and personal data. Attackers often disguise themselves as trusted organizations, banks, social media platforms, or government agencies.

The objective of this project is to analyze different email samples and identify phishing attempts by examining suspicious links, sender addresses, language patterns, and other red flags.

---

# 2. Project Objective

* Identify phishing emails and messages.
* Detect suspicious links and keywords.
* Analyze security threats.
* Create awareness about phishing attacks.
* Understand social engineering techniques used by attackers.

# 3. Sample Email Analysis

## Sample Email 1 – Fake Bank Alert

### Email Details

**From:** [security-update@sbi-secureverify.com](mailto:security-update@sbi-secureverify.com)

**To:** [rahul.sharma123@gmail.com](mailto:rahul.sharma123@gmail.com)

**Subject:** URGENT: Your SBI Account Will Be Suspended

**Message:**

Dear Customer,

We detected suspicious activity on your account. Your SBI account will be suspended within 24 hours unless you verify your details immediately.

Click below to verify:

[http://sbi-secureverify-login.com](http://sbi-secureverify-login.com)

Regards,
SBI Security Team

---

### Analysis

#### Red Flags

1. Creates urgency using "within 24 hours".
2. Sender email is not an official SBI domain.
3. Suspicious URL.
4. Generic greeting "Dear Customer".
5. Requests immediate action.

#### Why It Is Unsafe

The email attempts to steal banking credentials by directing users to a fake login page.

#### Verdict

**PHISHING EMAIL**

# 4. Common Phishing Indicators

| Indicator           | Description                  |
| ------------------- | ---------------------------- |
| Urgency             | Immediate action required    |
| Suspicious Links    | Fake websites                |
| Generic Greetings   | Dear Customer, Dear User     |
| Poor Grammar        | Spelling mistakes            |
| Fake Sender Address | Non-official domains         |
| Threats             | Account suspension warnings  |
| Unexpected Rewards  | Lottery, gift vouchers       |
| Credential Requests | Asking for passwords or OTPs |

---

# 5. Security Recommendations

1. Verify sender email addresses.
2. Never click suspicious links.
3. Check URLs before entering credentials.
4. Enable Multi-Factor Authentication (MFA).
5. Keep software updated.
6. Report suspicious emails.
7. Use anti-phishing protection tools.
8. Never share OTPs or passwords.

---

# 6. Red Flag Checklist

✅ Urgent language
✅ Unknown sender
✅ Suspicious URL
✅ Request for credentials
✅ Unexpected reward
✅ Threatening messages
✅ Poor grammar
✅ Generic greeting

If 2 or more indicators are present, treat the email as suspicious.
